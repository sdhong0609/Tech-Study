# Context란?
앱 환경의 글로벌한 정보에 대한 인터페이스로, 안드로이드 시스템에서 구현이 제공되는 추상 클래스입니다.<br>
앱의 현재 상태를 나타내며, 액티비티와 앱의 정보를 얻기 위해 사용할 수 있고, 리소스, 데이터베이스, Shared Preference 등에 접근하기 위해 사용할 수 있습니다.<br>
(* 참고: 여기서 인터페이스의 뜻은 "인터페이스란 서로 다른 두 개의 시스템 사이에서 정보나 신호를 주고 받는 상호작용을 지원하는 방법 또는 수단"이라는 의미로 사용되었습니다.)<br><br>

Context는 크게 두 가지로 나뉘는데, Application Context와 Activity Context입니다.<br>
애플리케이션 컨텍스트는 앱 전체의 라이프사이클을 가지며 싱글톤 인스턴스입니다.<br>
주로 현재의 컨텍스트와 분리된 라이프사이클을 가진 컨텍스트가 필요할 때, 혹은 액티비티의 범위를 벗어난 상황에서 컨텍스트가 필요할 때 사용됩니다.<br>
액티비티 컨텍스트는 액티비티 내에서 유효한 컨텍스트로, 액티비티 라이프사이클과 연결되어 있어서 액티비티가 종료될 때 컨텍스트도 소멸됩니다.<br>
액티비티 범위 내에서 컨텍스트를 전달하거나, 액티비티와 라이프사이클이 같은 객체를 생성해야 할 때 이 컨텍스트를 사용할 수 있습니다.
